ТЕХНИЧЕСКОЕ ЗАДАНИЕ

1. Общие сведения
Разработать веб-приложение "Дневник локаций" для сохранения и управления личными местами на интерактивной карте. Приложение должно работать в современных браузерах на ПК и мобильных устройствах. Бэкенд на Flask, фронтенд на нативном JavaScript, карта Leaflet, хранение в JSON файле.

2. Функциональные требования
Карта и маркеры: Отображение карты OpenStreetMap с возможностью масштабирования, перемещения и клика для добавления локаций. Маркеры имеют цветовую индикацию по категориям. При клике на маркер открывается попап с заметкой, датой, расстоянием до пользователя и кнопками редактирования/удаления.

Управление локациями: Добавление через клик по карте с заполнением заметки и категории в модальном окне. Редактирование заметки и категории. Удаление одной локации или всех сразу с подтверждением.

3. Нефункциональные требования
Технические: Python + Flask, HTML5/CSS3/JS, Leaflet 1.9.4, Font Awesome 6. JSON файл locations.json. Сервер на порту 5000 (host='0.0.0.0').

Производительность: Загрузка страницы до 2 секунд. Отклик на действия до 100 мс. Поддержка до 1000 локаций без деградации.
